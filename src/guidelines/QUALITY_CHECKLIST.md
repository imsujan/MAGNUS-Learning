### 1.1 Course-Level Learning Outcomes

- [ ] **Clear course outcomes defined** (3-5 specific outcomes)

- [ ] **Outcomes displayed prominently** in course description

**Quality Check:**

- [ ] **Each lesson has specific objectives** (1-2 per lesson)
- [ ] **Objectives use Bloom's Taxonomy action verbs** (Apply, Analyze, Evaluate, Create preferred)

- [ ] **Objectives are stated at the beginning** of each module/lesson

> Does each lesson clearly state what the learner will be able to DO after completing it?

- [ ] **Assessments align with learning objectives**

## ‚ôø Section 2: Accessibility Compliance

### 2.1 Video Accessibility

- [ ] **All videos have accurate captions** (human-reviewed, not just auto-generated)
- [ ] **Captions are properly synchronized** with audio
- [ ] **Captions follow best practices** (proper punctuation, speaker identification)
- [ ] **Downloadable transcripts provided** for all videos
- [ ] **Transcripts are formatted** for readability (paragraphs, headings)
- [ ] **Videos include audio descriptions** for critical visual-only content (or narrate actions)
- [ ] **No flashing content** that could trigger seizures (3Hz+ flashing)

**Quality Check:**
> Can a deaf/hard-of-hearing learner fully understand video content using only captions?

**Score:** ___/7

---

### 2.2 Visual Accessibility

- [ ] **Sufficient color contrast** (WCAG AA minimum: 4.5:1 for normal text, 3:1 for large)
- [ ] **Information not conveyed by color alone** (use icons, labels, patterns)
- [ ] **All images have descriptive alt text**
- [ ] **Charts/graphs have text descriptions** or data tables
- [ ] **Text is readable** (minimum 14pt in documents/slides)
- [ ] **Sans-serif fonts used** for body text (easier to read)
- [ ] **Text over images has sufficient contrast** (use backgrounds/overlays)

**Quality Check:**
> Can a colorblind learner distinguish all important visual information?

**Tools:** Use WebAIM Contrast Checker, Colorblind Web Page Filter

**Score:** ___/7

---

### 2.3 Navigation & Interaction Accessibility

- [ ] **Focus indicators visible** (can see which element is selected)
- [ ] **Skip navigation links provided** (for screen reader users)
- [ ] **No time-based restrictions** (or allow extension/pause)
- [ ] **Form labels are clear** and associated with inputs

**Quality Check:**
> Can a learner complete the entire course using only a keyboard (no mouse)?

**Score:** ___/6

- [ ] **Glossary provided** for technical terms
- [ ] **Content structured with headings** (H1, H2, H3 hierarchy)
- [ ] **Lists used for sequential information**
