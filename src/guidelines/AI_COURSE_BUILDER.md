### Prompt 1: Course Concept Development

```
I want to create an online course teaching [TOPIC] to [TARGET AUDIENCE] at a [BEGINNER/INTERMEDIATE/ADVANCED] level.
```

## Learning Objectives Generator

### Prompt 3: Course-Level Outcomes

```
Generate 3-5 course-level learning outcomes for a course on [TOPIC] targeting [AUDIENCE].

Requirements:
- Use action verbs from Bloom's Taxonomy levels 3-6 (Apply, Analyze, Evaluate, Create)
- Each outcome should describe a MEASURABLE skill
- Outcomes should align with real-world job requirements for [JOB ROLE]
- Include quality criteria or professional standards where applicable

Format each as: "By the end of this course, you will be able to [ACTION VERB] + [WHAT] + [HOW/CONTEXT] + [QUALITY STANDARD]"

Examples of strong action verbs:
- Apply: Demonstrate, Execute, Implement, Solve, Perform
- Analyze: Examine, Investigate, Differentiate, Diagnose
- Evaluate: Assess, Critique, Validate, Justify
- Create: Design, Build, Develop, Construct, Generate

Avoid weak verbs like: understand, know, learn about, be familiar with
```

```
Lesson context:
- Lesson duration: [X] minutes
- Key concept being taught: [CONCEPT]
- What learners already know: [PREREQUISITES]
- What they'll be able to do after: [OUTCOME]

Requirements:
- Use ONE action verb per objective (from Bloom's Taxonomy Apply-Create levels)
- Be specific about WHAT skill and HOW it will be demonstrated
```

## Lesson Outline Builder

### Prompt 5: Detailed Lesson Outline

```
Create a detailed lesson outline for a [X]-minute video lesson titled "[LESSON TITLE]" teaching [SPECIFIC CONCEPT/SKILL].

Learner context:
- Prior knowledge: [what they already know]
- Skill level: [Beginner/Intermediate/Advanced]
- Learning objective: [specific outcome]

Structure the outline with:

1. **INTRO (10% of time)**
   - Hook (why this matters, compelling example)
   - Learning objective statement
   - Overview of what will be covered

2. **MAIN CONTENT (70% of time)**
   - Break into 3-5 clear sections
   - For each section:
     * Section title
     * Key points to cover (3-5 bullets)
     * Demonstration steps (if applicable)
     * Visual aids needed (screenshots, diagrams)
     * Common mistakes to highlight
   - Include "pause and practice" prompts where appropriate

3. **PRACTICAL EXAMPLE (15% of time)**
   - Real-world scenario demonstrating the concept
   - Step-by-step walkthrough
   - Expected results

4. **CONCLUSION (5% of time)**
   - Recap key takeaways (3-4 points)
   - Preview next lesson
   - Call to action (what to practice)

Include timing estimates for each section. Ensure content flows logically and builds understanding progressively.
```

```
B) Document clashes in a report and send to the project manager
C) Schedule a coordination meeting with the MEP engineer to discuss alternative routing
D) Raise ceiling heights to create more clearance

**Correct Answer:** C

**Explanation:** [Detailed reasoning about why C is the best professional approach, and why A, B, D are inadequate]

Make scenarios realistic and representative of actual job challenges in [INDUSTRY/ROLE].
```

## Best Practices for Using AI Prompts

### 1. Prompt Engineering Tips

**Be Specific:**
‚ùå "Create quiz questions for my course"
‚úÖ "Generate 10 multiple-choice questions testing understanding of Revit wall types, including 2 scenario-based questions, for intermediate learners"

**Provide Context:**
- Always include target audience, skill level, topic
- Specify learning objectives or outcomes
- Mention prerequisites or prior knowledge
- State the purpose (formative assessment vs. summative)

**Iterate:**
1. Run initial prompt
2. Review output
3. Refine prompt with additional requirements
4. Regenerate improved version

**Request Formats:**
- Specify desired output format (bullet list, table, script, etc.)
- Include word count or time limits
- Request specific structure or sections

---
